# PET/CT 患者友好图片人工评价方案

**版本：** 2.0  
**日期：** 2026-06-21  
**状态：** 工作草案，允许在新实验 ID 中继续迭代  
**对应 SAP：** `docs/statistical_analysis_plan.md` 2.0

## 评价对象

同一批 300 例报告分别产生一张 `UNGATED` 最终图片和一张 `GATED` 最终图片，共 600 个评价对象。人工评价只比较图片与原始报告，不以 AI 门控结果作为依据。

## 整图判定

- `PASS`：图片可用于患者解释；报告中的关键病灶、侧别、位置、性质、FDG/SUVmax 和重要文字均正确，且没有会误导患者的新增事实。
- `FAIL`：存在任何可能影响患者理解或临床安全的错误。选择 `FAIL` 时必须勾选至少一个错误类型。

主要研究采用二元整图判定，而不是旧方案中的 5 分 Likert 总分。Likert 评分如后续开展，只能作为独立探索性研究，不能与当前主要终点混合。

## 固定错误分类

| 编码 | 定义 |
|---|---|
| `LATERALITY` | 患者左右侧、成对器官侧别或引导线落点错误 |
| `CHINESE_TEXT` | 中文乱码、错字、残缺或标签指代错误 |
| `LESION_LOCATION` | 病灶解剖分区或引导线定位错误 |
| `OMISSION` | 报告中的关键病灶或关键信息未呈现 |
| `ANATOMICAL_DISTORTION` | 器官形态、位置或层次关系明显不合理 |
| `BENIGN_MALIGNANT` | 性质倾向或颜色编码与报告不符 |
| `SUVMAX` | SUVmax/FDG 描述遗漏、数值错误或病灶错配 |
| `STYLE` | 严重影响患者阅读的版式、信息密度或风格问题 |
| `HALLUCINATION` | 新增报告中不存在的病灶、异常、数值或结论 |
| `OTHER` | 以上分类无法覆盖，必须在备注中说明 |

一张图片可记录多个错误类型。正式批次开始后不得修改分类定义。

## 评价流程

1. 核对病例编号和原始报告；
2. 从报告中识别关键发现；
3. 逐项核对图片中的器官、病灶点、引导线、文字、颜色和数值；
4. 判断是否存在遗漏或新增事实；
5. 保存整图 `PASS/FAIL`、错误类型、评价者和备注。

评价者应尽可能不参考 AI 门控结论。若存在多名评价者，须独立评分；分歧应按研究方案形成最终仲裁结果，并明确保留原始评分。

## 数据完整性

正式分析前，600 个运行必须全部具有最终人工判定。不得将未评价、API 失败或找不到文件的运行自动视为通过，也不得在分析脚本中用模拟数据补齐。
