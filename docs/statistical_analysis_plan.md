# PET/CT 患者友好可视化回顾性配对研究统计分析计划（SAP）

**版本：** 2.0  
**日期：** 2026-06-21  
**状态：** 工作草案，尚未投稿或最终锁库；开始某个正式分析批次前需由研究团队确认  
**适用研究：** 三癌种 300 例、同病例 `UNGATED` 与 `GATED` 配对验证  
**替代版本：** 本文件完全替代 2026-02-04 的“2100→1500、三阶段评价”SAP。旧版设计不再用于当前主要研究。

## 1. SAP 是什么

SAP 是 Statistical Analysis Plan，即统计分析计划。它在正式查看研究结果前规定研究终点、分析人群、统计方法、缺失数据处理和亚组分析规则，避免看到结果后再选择有利的分析方法。

## 2. 研究目的与设计

本研究为回顾性、同病例配对的生成性能验证。纳入肾癌、前列腺癌和尿路上皮癌各 100 例，共 300 例。每例报告在冻结的模型、提示词、参考图和生图参数下分别运行：

- `UNGATED`：仅生成一次，第一张图片直接进入人工评价；
- `GATED`：生成后由视觉模型结合原始报告审查，不通过时根据审查意见进行局部修订，直至通过或达到冻结的最大修订次数。

分析单位是病例报告。两种策略使用相同报告，通过 `pairing_key` 配对。批量运行顺序及每个病例内两种策略的先后顺序由冻结的随机种子决定。

## 3. 研究人群与数据锁

正式分析集必须同时满足：

1. 三癌种各 100 例；
2. 每个 `pairing_key` 恰有一个 `UNGATED` 和一个 `GATED` 结果；
3. 600 个计划运行均完成；
4. 每个结果均有医生 `PASS` 或 `FAIL` 评价；
5. 实验 ID、模型、图像参数、提示词版本、代码源树哈希和随机种子在批次内一致。

基础设施/API 短暂失败应通过断点续跑解决，不得静默删除病例或更换样本。存在未解决运行时不得执行正式主要分析。无缺失值插补。

正式批次由 `experiment_lock.json` 锁定：

- 300 例队列及每份报告 SHA-256；
- 原始队列文件 SHA-256；
- 600 个配对工作项及执行顺序；
- 生图、审查和出题提示词版本及 SHA-256；
- provider、模型、参数、最大修订次数及参考图 SHA-256；
- 随机种子、代码源树 SHA-256、Git commit/dirty 状态。

当前图像 API 没有可用的 seed 参数。因此随机种子只保证病例与策略执行顺序可复现，不能保证图像像素级复现；记录中的 `providerSeedApplied=false` 必须在论文局限性中说明。

## 4. 人工金标准

医生依据原始 PET/CT 报告评价每种策略最终返回的图片：

- `PASS`：关键报告事实均正确呈现，且不存在足以误导患者的错误；
- `FAIL`：至少记录一个结构化错误类型。

错误类型固定为：

`LATERALITY`、`CHINESE_TEXT`、`LESION_LOCATION`、`OMISSION`、`ANATOMICAL_DISTORTION`、`BENIGN_MALIGNANT`、`SUVMAX`、`STYLE`、`HALLUCINATION`、`OTHER`。

AI 门控判断不是准确性金标准。正式分析只使用医生评价。若采用多评价者，须独立评分并在数据库锁定前形成预先规定的最终仲裁结果；主要分析使用最终仲裁后的二元结论。

## 5. 主要终点

主要终点是 `UNGATED` 策略首次生成图片的医生 `PASS` 比例：

`UNGATED PASS 率 = UNGATED PASS 病例数 / 300`

报告点估计及 Wilson 95% 置信区间。预设性能目标为点估计不低于 95%。判定规则固定为：

`UNGATED PASS 率点估计 ≥ 0.95`

95% 置信区间用于表达估计不确定性，不作为该项目内部 Go/No-Go 阈值。论文中不得将“点估计达到 95%”表述成“95% 置信区间下限达到 95%”。

## 6. 关键次要终点

比较同一病例 `GATED` 与 `UNGATED` 的医生 `PASS/FAIL`：

- 报告两种策略通过率；
- 报告配对通过率差 `GATED − UNGATED`；
- 使用双侧精确 McNemar 检验；
- 报告不一致配对数：
  - `UNGATED FAIL / GATED PASS`；
  - `UNGATED PASS / GATED FAIL`；
- 配对率差的 95% CI 使用病例级配对 bootstrap，10,000 次抽样，固定种子 `20260621`。

显著性水平为双侧 `α=0.05`。该比较为预设的关键次要分析。

## 7. 其他次要与探索性分析

- 按癌种分别报告通过率、Wilson 95% CI 和精确 McNemar 检验；
- 按策略统计各错误类型的病例数和每运行发生率；
- 描述门控修订次数、首次通过比例、达到上限仍不通过比例；
- 描述运行耗时；
- 将 AI 最终门控失败视为“阳性”、医生最终图片 `FAIL` 为参考，计算敏感度、特异度、PPV 和 NPV；
- 如开展额外模型、提示词或参数比较，必须使用新的实验 ID，并明确标记为独立或探索性批次。

癌种分层和错误谱分析为探索性，不进行多重比较校正，不据此作确定性优效结论。

## 8. 分析实现与产物

正式输入由以下命令生成：

```powershell
python scripts/experiments/export_paired_results.py `
  --experiment-id <experiment_id> `
  --output data/analysis/paired_runs.csv
```

正式分析命令：

```powershell
python scripts/analysis/run_statistical_analysis.py `
  --input data/analysis/paired_runs.csv `
  --output-dir data/analysis/results
```

脚本不包含模拟数据回退。任何完整性或冻结条件不满足时均退出并返回错误。主要产物为：

- `analysis_results.json`：主要结论及输入文件 SHA-256；
- `pass_rates.csv`：总体和癌种分层通过率及 Wilson 95% CI；
- `mcnemar.csv`：配对列联表、率差、bootstrap CI 和精确 P 值；
- `error_types.csv`：错误类型分布。

所有论文数值必须能够追溯至这些输出文件和对应的 `experiment_lock.json`。

## 9. 方案偏离

正式实验开始后不得在同一实验 ID 下更改队列、模型、提示词、参考图、参数、最大修订次数、随机种子或代码。任何必要变更应：

1. 停止当前实验；
2. 在决策记录中写明原因；
3. 使用新实验 ID 和新锁文件；
4. 将旧批次保留为方案偏离或探索性结果，不与新批次混合。
