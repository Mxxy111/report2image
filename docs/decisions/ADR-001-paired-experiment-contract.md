# ADR-001：采用不可变实验锁和追加式审计日志

## 状态

Accepted

## 日期

2026-06-21

## 背景

正式研究需要对同一批 300 例分别运行无门控和有门控策略。旧的单病例 Web 记录不足以证明批次中途没有更换样本、模型、提示词或参数，也无法可靠地从长时间 API 任务中断处继续。

## 决策

每个正式批次使用独立 `experiment_id`，并在第一次准备时创建：

- `experiment_lock.json`：冻结队列、文件哈希、模型参数、提示词、随机种子和代码状态；
- `work_items.csv`：冻结 600 个配对工作项及执行顺序；
- `events.jsonl`：以追加方式记录锁定、会话、开始、完成、失败和恢复事件。

已完成工作项在恢复时自动跳过。若图片与 manifest 已保存但完成事件尚未来得及写入，恢复逻辑从 manifest 重建完成事件。锁文件与当前环境不一致时拒绝继续，必须使用新的实验 ID。

## 后果

- 正式批次可断点续跑且不会静默重复已完成工作；
- 论文中的每个结果可追溯到病例、运行、提示词版本和代码哈希；
- 正式运行开始后不能在同一实验 ID 下修复代码或调整提示词；
- 图像 API 不支持 seed，因此只能复现执行顺序，不能保证像素级确定性。

## 被替代的做法

- 手工逐例运行后再拼接结果；
- 仅按文件创建时间判断是否完成；
- 找不到真实统计输入时自动生成模拟结果；
- 使用旧版“2100→1500、三阶段评价”作为当前主要研究方案。
