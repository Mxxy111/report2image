**版本：** 0.1 工作草案  
**日期：** 2026-06-21  
**状态：** 用于统一研究主线和系统需求，尚非伦理递交稿、临床试验注册稿或投稿终稿  

## 一、课题名称

中文题目建议：

**基于生成式人工智能的泌尿系统肿瘤 PET/CT 报告患者友好可视化：回顾性准确性验证与门诊应用研究**

英文题目建议：

**Generative AI-Based Patient-Friendly Visualization of PET/CT Reports in Urologic Malignancies: A Retrospective Accuracy Validation and Outpatient Application Study**

## 二、研究背景与立项依据

PET/CT 报告通常包含多器官、多病灶、多数值和多种不确定性表达。对于泌尿系统肿瘤患者，报告中常涉及原发灶、术后改变、淋巴结、骨转移、肺部结节、炎症、良性伴随病变、FDG 或 PSMA 摄取以及 SUVmax 等信息。这类报告对临床医生和患者都有较高阅读负担，尤其在门诊场景中，医生需要在有限时间内向患者解释病灶部位、良恶性倾向和后续建议。

生成式图像模型和多模态大模型为“把医学报告转化为患者可理解信息图”提供了新的技术可能。与传统影像重建不同，本课题并不试图复刻 PET/CT 图像，而是将报告中已经形成的文字结论转换为简洁、温和、患者友好的二维医学信息图。该工具如果能够稳定、准确地表达报告事实，有望改善医患沟通效率，并为患者教育材料生成提供新路径。

但此类应用具有明确风险。生成式模型可能出现病灶遗漏、幻觉、左右侧错误、病灶定位错误、性质或颜色编码错误、SUVmax 错配、中文标签错误和解剖结构不合理等问题。若直接用于门诊解释，错误图像可能放大患者误解。因此，在进入真实临床使用前，必须先完成可复现的回顾性准确性验证，并明确 AI 质量门控能否降低错误。

本课题的核心定位是：先评价生成式 AI 将 PET/CT 报告转化为患者友好图片的技术准确性和安全边界，再探索其在门诊沟通中的临床效用。当前阶段聚焦第一部分，即三癌种 300 例回顾性配对验证。

> 文献综述与国内外现状需要后续单独补齐。正式文本中应补充医学报告患者友好化、放射报告可视化、生成式 AI 医学文本/图像应用、多模态质量控制和患者教育材料评估等方向的证据，不在本草案中虚构引用。

## 三、总体研究目标

本课题拟建立一套可审计、可复现的 PET/CT 报告患者友好可视化生成流程，并评价其准确性、安全性和潜在门诊应用价值。

具体目标包括：

1. 构建一个可配置的 PET/CT 报告可视化生成系统，支持模型配置、参考图、AI 审查、失败反馈修订、人工评价和运行记录。
2. 在肾癌、前列腺癌和尿路上皮癌三类泌尿系统肿瘤报告中，评价无门控一次生成图片的医生整图通过率。
3. 比较同一病例在无门控策略和 AI 门控修订策略下的最终图片通过率，判断 AI 质量门控是否带来增益。
4. 分析主要错误类型，识别影响临床可用性的关键失败模式。
5. 为后续门诊随机应用研究建立数据结构、评价流程和安全边界。

## 四、核心研究问题

当前阶段需要回答以下问题：

1. 生成式图像模型能否将泌尿系统肿瘤 PET/CT 报告稳定转换为患者友好医学信息图？
2. 无 AI 门控时，一次生成图片的医生整图通过率能否达到预设性能目标？
3. 对同一病例加入 AI 图文审查和反馈修订后，最终图片通过率是否优于无门控一次生成？
4. 失败图片主要集中在哪些错误类型，例如左右侧、病灶遗漏、幻觉、SUVmax 错误或解剖定位错误？
5. AI 门控对医生金标准的敏感度和特异度如何，是否适合作为前置质量控制工具？

后续门诊阶段拟回答：

1. 与传统文字报告相比，AI 患者友好图片是否提高患者理解度、满意度和记忆准确率？
2. AI 图片是否缩短医生解释报告的时间？
3. 门诊真实使用中，医生是否认为图片可用、需修改或存在误导风险？

## 五、研究假设

当前回顾性阶段的主要假设为：

1. 生成式图像模型可以在多数病例中生成与 PET/CT 报告关键事实一致的患者友好图片。
2. 无门控一次生成图片的医生整图通过率可达到或接近预设目标。
3. AI 质量门控与反馈修订能够减少部分可识别错误，使同一病例的最终通过率高于无门控策略。

需要强调的是，AI 门控结果不是金标准。本研究所有准确性结论均以医生人工评价为准。

## 六、研究设计

本课题分为两个阶段，当前优先执行第一阶段。

### 第一阶段：回顾性配对准确性验证

研究类型：回顾性、同病例配对、技术准确性验证研究。

研究对象：历史 PET/CT 或相关核医学报告，覆盖肾癌、前列腺癌和尿路上皮癌。

样本设计：

- 肾癌 100 例；
- 前列腺癌 100 例；
- 尿路上皮癌 100 例；
- 合计 300 例。

每个病例分别运行两种策略：

- `UNGATED`：报告输入后生成一次图片，不经过 AI 审查；
- `GATED`：报告输入后生成图片，视觉 AI 同时读取原报告和生成图片进行审查；若不通过，则根据具体错误反馈进行有限次数修订。

两种策略的执行顺序由固定随机种子决定，以减少顺序相关偏倚。每个病例使用相同报告文本、同一批次冻结的模型、提示词、参考图和参数。

### 第二阶段：门诊应用与患者理解度研究

该阶段尚未启动，暂作为后续计划。拟采用门诊前瞻性研究设计，比较传统文字报告与“文字报告 + AI 患者友好图片”对患者理解度、满意度和医生解释时间的影响。

第二阶段启动前需另行完成：

- 正式临床试验方案；
- 样本量估算；
- 随机化方法；
- 纳入和排除标准；
- 患者知情同意；
- 伦理审批；
- 数据安全方案；
- 量表选择和答题流程。

## 七、研究对象与数据来源

### 数据来源

当前回顾性阶段使用既往积累的泌尿系统肿瘤核医学报告数据。现有数据集中包含：

- 原始 CSV 报告；
- 三癌种各 100 例的评测队列；
- 对应报告文本；
- 历史图片和错误病例资料。

正式实验输入队列位于：

```text
data/derived/evaluation_dataset/
```

实验运行时会生成实验锁文件，记录输入文件 SHA-256、病例配对键、报告 SHA-256 和每个工作项顺序。

### 纳入标准（工作草案）

1. 诊断或临床背景属于肾癌、前列腺癌或尿路上皮癌相关病例；
2. 报告包含可用于生成患者友好图片的核医学检查文字；
3. 报告中至少包含检查所见或检查结论；
4. 病例号或研究编号可用于配对和追踪；
5. 抽样后能够形成三癌种各 100 例的固定队列。

### 排除标准（工作草案）

1. 报告文本严重缺失或无法解析；
2. 病例编号缺失，无法建立配对键；
3. 同一报告重复出现且无法区分；
4. 报告内容与目标任务明显不相关；
5. 数据使用权限或脱敏条件不满足研究要求。

正式纳排标准需要在最终实验前进一步确认。

## 八、技术路线

### 1. 报告输入与标准化

系统读取病例编号、检查所见和检查结论，组合为模型输入。对每份报告生成稳定配对键，用于连接无门控和有门控结果。

### 2. 患者友好图片生成

生成提示词要求模型只使用报告原文作为医学事实来源，输出简洁二维信息图，不复刻 PET/CT 原始影像，不添加报告未提及的病灶、数值或结论。

关键约束包括：

- 只绘制报告涉及器官和必要定位结构；
- 严格处理患者左右侧别；
- 保留 FDG/PSMA 摄取和 SUVmax；
- 区分恶性、疑似恶性、良性或炎症倾向；
- 使用简体中文患者友好标签；
- 不生成报告外病灶或结论。

### 3. AI 视觉门控

门控模型同时读取原始报告和生成图片，输出通过或不通过。不通过时需给出具体修图要求。门控重点包括：

- 病灶是否遗漏；
- 是否新增报告外异常；
- 左右侧别是否正确；
- 病灶定位和引导线是否正确；
- 颜色和良恶性倾向是否服从报告；
- SUVmax 是否准确对应；
- 中文标签是否可读；
- 解剖结构和患者友好风格是否合格。

### 4. 反馈修订

若门控失败，系统使用上一张图片作为编辑基础，按门控反馈进行局部修订。最大修订次数在实验锁中固定。

### 5. 人工评价

医生依据原报告对最终图片进行整图 `PASS/FAIL` 判断。失败时记录错误类型。人工评价是准确性金标准。

### 6. 可审计批量运行

批量实验生成：

```text
runtime/experiments/<experiment-id>/
├─ experiment_lock.json
├─ work_items.csv
└─ events.jsonl
```

单次运行结果保存：

```text
runtime/cases/<日期>/<病例号>/<run_id>/
├─ attempt_1.png
├─ attempt_2.png
├─ reference.png
└─ manifest.json
```

## 九、干预与对照

本研究的“干预”不是对患者实施治疗，而是对同一份报告采用不同生成策略。

对照策略：

`UNGATED`，一次生成，无 AI 审查。

实验策略：

`GATED`，生成后进行 AI 质量门控，失败时根据反馈修订。

除门控与修订流程外，两组应使用同一报告、同一模型批次、同一提示词版本、同一参考图和同一生图参数。

## 十、结局指标

### 主要结局

`UNGATED` 一次生成图片的医生整图通过率。

判定规则：

```text
UNGATED PASS 率 = UNGATED PASS 病例数 / 300
```

当前工作草案中的性能目标为点估计不低于 95%。Wilson 95% 置信区间用于表达不确定性。

### 关键次要结局

同一病例中 `GATED` 与 `UNGATED` 的医生整图通过率差异。

分析内容：

- `GATED` 通过率；
- `UNGATED` 通过率；
- 配对通过率差；
- `UNGATED FAIL / GATED PASS` 数量；
- `UNGATED PASS / GATED FAIL` 数量；
- 精确 McNemar 检验。

### 其他次要结局

1. 各错误类型发生率；
2. 癌种分层通过率；
3. 门控修订次数；
4. 首次门控通过率；
5. 达到最大修订次数后仍失败比例；
6. AI 门控与医生金标准的一致性；
7. 生成耗时和失败原因；
8. 不同模型或提示词版本的探索性比较。

## 十一、错误分类

正式错误分类固定为：

- `LATERALITY`：左右侧错误；
- `CHINESE_TEXT`：中文标签错误；
- `LESION_LOCATION`：病灶定位错误；
- `OMISSION`：关键病灶或信息遗漏；
- `ANATOMICAL_DISTORTION`：解剖结构畸变；
- `BENIGN_MALIGNANT`：性质或颜色编码错误；
- `SUVMAX`：SUVmax 或 FDG/PSMA 信息错误；
- `STYLE`：患者友好性或版式问题；
- `HALLUCINATION`：报告外新增医学事实；
- `OTHER`：其他错误。

正式批次开始后不得在同一实验 ID 内修改错误分类。

## 十二、统计分析计划

正式统计分析以 `data/analysis/paired_runs.csv` 为输入，脚本不允许使用模拟数据补齐。

主要分析：

- 计算 `UNGATED` 通过率；
- 报告 Wilson 95% 置信区间；
- 判断点估计是否达到预设性能目标。

配对分析：

- 使用双侧精确 McNemar 检验；
- 报告配对通过率差；
- 使用病例级配对 bootstrap 估计率差 95% CI。

分层和探索性分析：

- 按癌种分层；
- 按错误类型统计；
- 分析门控诊断性能；
- 描述耗时和修订次数。

正式统计方案见 `docs/statistical_analysis_plan.md`。若研究主线改变，例如加入第三种策略或新模型正式比较，必须同步更新 SAP。

## 十三、质量控制与可复现性

每个实验 ID 形成不可变批次快照。记录内容包括：

- 原始队列文件 SHA-256；
- 报告 SHA-256；
- 600 个工作项及随机顺序；
- 随机种子；
- provider ID、API 类型、Base URL、模型和超时；
- 图像参数；
- 参考图 SHA-256；
- 提示词版本和 SHA-256；
- SAP 和人工评价方案 SHA-256；
- 代码源树 SHA-256；
- Git commit 和 dirty 状态；
- 每次运行的开始、完成、失败和恢复事件。

需要说明的是，当前图像 API 不支持 seed 参数。随机种子用于病例顺序和策略顺序，不保证图像像素级完全复现。

## 十四、数据安全与伦理考虑

本课题涉及病例号、姓名、检查报告等敏感信息。当前开发阶段已将 `.env`、本地 provider 配置、`runtime/` 和 `outputs/` 排除出版本提交，但真实临床研究前仍需完善：

1. 数据脱敏规则；
2. 访问控制；
3. 操作日志；
4. 数据保存期限；
5. 备份与销毁流程；
6. 伦理审批；
7. 知情同意或豁免知情同意依据；
8. 多评价者评分和仲裁记录的权限管理。

AI 生成图片不得替代医生诊断，不得作为治疗决策依据。当前定位为患者沟通辅助材料和研究工具。

## 十五、当前系统需求边界

为了防止主线发散，当前版本优先满足以下需求：

1. 单病例网页生成和人工评价；
2. 三癌种 300 例配对队列；
3. 无门控与有门控两种策略；
4. 可恢复批量运行；
5. 可追溯元数据；
6. 严格真实数据统计；
7. 工作版研究文档。

当前暂不纳入主线的需求：

1. 多策略正式比较；
2. 门诊随机分组；
3. 患者答题结果采集；
4. 满意度和理解度量表；
5. 多评价者盲法工作台；
6. 自动发表级图表生成；
7. 全流程脱敏和权限系统。

这些内容可以作为后续扩展，但不应阻塞当前回顾性配对验证。

## 十九、主要风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| 模型幻觉或病灶遗漏 | 患者误解风险 | 医生人工评价作为金标准，记录错误谱，不直接临床替代 |
| 左右侧错误 | 高风险误导 | 提示词和门控均加入左右侧强规则 |
| API 不稳定 | 批量运行中断 | 追加式审计日志和断点续跑 |
| 中途更换模型或提示词 | 破坏可比性 | 每个实验 ID 使用不可变实验锁 |
| 统计数据不完整 | 无法正式分析 | 分析脚本强制检查完整性 |
| 数据隐私风险 | 伦理和合规问题 | 真实门诊前补齐脱敏、权限和伦理 |
| 方案持续膨胀 | 主线混乱 | 当前阶段只聚焦 300 例配对验证 |

## 二十、当前需要研究团队确认的问题

1. 课题主标题是否聚焦“患者友好可视化”，还是更强调“AI 质量门控”？
2. 当前主要终点是否继续使用 `UNGATED` 一次生成通过率？
3. 95% 目标是采用点估计达到 95%，还是要求置信区间下限达到 95%？
4. 三癌种各 100 例是否足够，是否需要样本量论证或扩大样本？
5. 医生评价采用单评价者、双评价者还是三评价者？
6. 是否需要盲法隐藏策略信息？
7. 是否将门诊患者理解度研究纳入同一课题，还是作为第二课题？
8. 是否需要在当前阶段加入其他模型正式比较？
9. 是否需要先做文献综述和注册方案，再运行最终批次？
10. 数据脱敏和伦理审批由谁负责、何时启动？

## 二十一、工作结论

本课题当前最清晰、最可执行的主线是：先完成三癌种 300 例回顾性配对准确性验证，用医生整图通过率作为金标准，评价无门控一次生成能力和 AI 门控修订增益；在该基础上，再决定是否进入门诊前瞻性应用研究。所有新模型、新提示词和新策略比较都应使用独立实验 ID，并在进入正式比较前更新本计划书和 SAP。
